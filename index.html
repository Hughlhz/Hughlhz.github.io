写在前面

考研成绩

  • 复试 总分240，综合面试161，英语79 最终排学硕第九

  • 高数和线代有一定基础
  • 熟悉英语语法

我本科是电子科大软件学院，能拿得出手的奖项只有中国高校计算机大赛网络技术挑战赛的国二，其他都是些文体比赛，什么“方言朗读者大赛“、“万米接力赛“等等。

由于学院对实习的强制要求（二月份到八月份），我只能从八月份之后开始复习，真正复习开始时间是九月初返校后，之前几乎没有复习过任何科目。

这是我在八月份结束实习后的计划，本来计划8月22号就返校开始复习，但最后还是拖延到了9月4号才返校。

所以我总共的复习时间在103天左右。

决定考研后的第一步是要快速了解各个科目的特点，因地制宜地去复习，特别推荐加入王道电子科技大学考研群，里面资料全，比外面机构卖的资料还全，而且质量好，初试复试资料都有。

数学一内容多，难度最大，我计划进行三轮复习，第一轮把基础捡回来，第二轮强化并刷真题，第三轮做模拟题，查缺补漏；

英语呢，我感觉如果语法基础很好，那么只需要背考纲词汇以及刷近十年的真题即可，作文也只需要根据近十年的真题去准备；

政治只有马原需要强化理解，其他科目记忆（后面会讲记忆方法）即可，最后主要是靠刷客观题，我政治客观题大概是42-44分；

专业课是820，一定要按照考纲去复习，注意820不考串这个知识点，刷题刷往年真题和王道的课后习题就足够了（看到有大佬提到剑指offer，里面题目确实和820的代码提很像，我考前也翻阅过）。

针对各科目需要记忆，或者说容易遗忘的知识点，我建议自己手写整理起来，后期复习这些整理的笔记，又快又好，最好建立起知识点和笔记对应位置这样的映射。到后期，只需看笔记即可实现对关键知识点一轮复习，省时高效。做笔记的过程很痛苦，但是用的时候非常爽。

数学

我踩过的坑

数学我看过的参考书主要是张宇老师的基础三十讲，强化三十六讲，武忠祥老师的高数讲义，李正元的全书（除了武忠祥的高数讲义，其他都是白嫖保研的hxd的 >–<），上过的网课主要是张宇的配套课程，武忠祥的配套课程，还有一部分概率论网课（没感觉哪个老师讲得特别好）。习题册一本没刷，所以我做的题目主要就是各辅导书上的例题、往年真题以及最后各机构出的模拟题。

推荐路线

我个人的建议是，基础阶段可以用张宇的基础三十讲以及配套的视频，强化阶段高数建议武忠祥的高数讲义及配套视频，线代和概率用张宇的线代九讲和概率九讲以及配套的视频，如果张宇概率论不够的话，可以刷一下李林的概率论部分。真题用世纪金榜的近几年真题即可，冲刺时用余炳森5套卷、合工大共创、合工大超越、武忠祥押题卷（只有一套卷）、李林4套卷、李林6套卷，不要去写李艳芳的模拟卷。用IPad+ES文件浏览器+百度云盘+PDF expert观看视频、电子书效果最佳，在ES文件浏览器里登录百度云盘，可以无会员倍速播放。

Q&A：

  • 为什么不听张宇的高数强化课？

    因为张宇的这本高数十八讲不是按照高数课本的结构划分的，而是按张宇自己对题型的理解划分的，这点和我之前学高数的框架冲突，果断放弃，而相比武忠祥老师，他的讲义是按照高数课本的结构划分的，符合我的学习习惯，而且武老师能一针见血地指出“经典错误标准零分”，我现在还记忆犹新。

  • 为什么不写李艳芳模拟卷？

    因为那套卷子难度太大，和考研的难度相去甚远，纯粹为了难而难，很搞心态。

  • 为什么不刷习题册？

    习题册我也有，保研室友有张宇的一千题（数一），我一开始准备考中大买了一千题（数二），但是都由于时间不够，我果断放弃，选择把辅导书上的例题理解透彻。

  • 需要做错题集吗？

    时间不够做错题集，建议直接在辅导书上用不同记号标注重点题、易错题。

政治

我踩过的坑

由于时间紧张，我只上了徐涛的马原强化班网课，配套书是徐涛的黄皮书，一千题也只刷了马原部分（而且马原部分还没刷完），冲刺用的是徐涛和腿姐的冲刺笔记，模拟卷主要有肖四肖八、腿姐、米鹏，只做客观题。今年踩过最大的坑应该是肖老师了，肖老师客观题难度和真题很接近，但是大题几乎没中，我客观题参照市面上的答案是42-44左右，最后只有74分。大题不建议花大量时间背诵，听说合理地抄材料分数也挺高的。

英语

我踩过的坑

语法基础好的情况下背考纲词汇即可。我在背考纲词之前，阅读做五个错五个，一度怀疑人生，背了之后就好很多，一套卷大概错两到三个。刷题本来计划刷近十年真题，但是最后临近考试才发现做不完了，最后只做了2010年-2015年和2021年的真题的阅读理解、翻译，作文是选取了近五年真题中出现过的几类有代表性的题型去复习。建议大家做题一定要按照一个固定的做题顺序去做，我就是因为考前几乎没有做过一整套完整的卷子，然后上来就写翻译题，导致只剩5分钟写完形填空，很亏！

推荐路线

考纲词是一定要背的，我推荐IPad+List背单词+List生词本，安卓的话扇贝单词就好了。两个都是付费软件（加起来十元左右吧），可以配合使用，非常好用。List背单词里收录了最新考纲词汇，根据自己需求划分list，每天会有不同的记忆list和复习list。然后自己觉得有必要多轮复习的单词，可以添加到List生词本中（两个App联动使用），按照你的意愿重新划分list。背单词的界面也非常简洁，而且功能强大，可以发音、查词，隐藏中文或英文，按照熟练度排序等等，也有听写模式和拼词模式（可以用键盘输入），非常好用，墙裂推荐，效率爆表！

背单词建议用大块的时间，比如早上起来的两小时，复习单词建议用碎片时间，路上、食堂都可以，这就要求你的生词本要划分得够小，我用List生词本按开头字母划分生词，每个list大概50-80个词这样，碎片时间基本上够用。

完形填空和翻译不建议做过多练习，收益不高。

作文一定要自己总结，感觉这作文就跟八股文一样，一板一眼，有定式，但是对于不同的题材，又有不同的套法，最好就是找几类近年的真题作为模板，以不变应万变。

专业课820

我踩过的坑

踩过的坑都是在进王道群之前踩的，进了王道群电子科大群基本就没坑了，里面各路大神指路，专业课整理得非常好，感谢各路大大！复试资料在前王道群才有，不要迷路了！

推荐路线

b站上有王道讲课的视频，看的时候很方便，配合王道课本、课后习题使用就够了。OS的内容需要理解，因此我建议做好笔记并进行多轮复习（做的笔记要突出重点，而且能提纲挈领），到后期用背诵的形式复习；DS的代码题的话，二叉树和线性表一定要看，学有余力建议看剑指Offer，再有余力可以看些算法思想，如动态规划等，当然做到前两个基本就够了，然后像一些二叉树的性质、算法复杂度分析等需要理解的知识，也是建议做好笔记并进行多轮复习。

时间安排

总共就一百来天的时间，时间安排很重要。

劳逸结合

首先要强调的一点是劳逸结合，保证睡眠时间和睡眠质量。睡眠时长我控制在七个半小时+半小时以内的午休+（偶尔）半小时以内的晚饭后的小睡，晚上十二点睡觉，早上七点半左右起床，这样看睡眠时间是很充足的。睡眠质量的保证我是通过锻炼来保证的，每天在晚饭前做定量的俯卧撑、引体向上和双杠臂屈伸，这样不仅可以很好地提高睡眠质量，还可以锻炼肌肉（至少我背肌在当时是练出一点来了的）。另外不带手机上床睡觉，帮助也很大。

适量的游戏可以激活大脑，因此进入十二月之前我还是有打游戏，但是在进入十二月后，建议暂停游戏，全力复习。并且建议不要每天都安排游戏时间，而是固定每周的某天下午或晚上玩，个人感觉这样收益比较高。

学习安排

复习时间只有3个多月，因此学习周期不能再按月划分了，我是按照3天x12小时/天小周期，7天x12小时/天一个大周期这样复习，每天早八到图书馆，晚上十点二十闭馆回宿舍，有时候回宿舍还想学的话还要再学一会。

十月份之前主要是背单词+学数学。因此除了早上起来背两小时单词，加碎片时间复习单词，其他时间基本上都在学习数学，基础课难度不大，因此我没刷视频，而是对着张宇的基础三十讲硬啃，要求是把基础的概念理解，了解下大体结构即可。基础课看完之后，就要立刻开始强化班。上强化班你要做的事情就是把老师讲义上的重点题目吃透，要对讲义上的题目根据你自己的熟练程度做不同的标记，以方便二刷、三刷。视频建议只听一遍，强化课讲义根据你自己的情况选择性多刷。第一遍强化我每个小科目都是花大概7天x12小时/天的时间，如高数上、高数下、线代、概率，二刷时间缩减为3天x12小时/天，三刷则往往一天甚至半天就要看完。在强化完后，就可以做做真题检验水平，同样的道理，真题的错题也要标注好，以便二刷。

在结束数学一轮强化后，我开始选择性复习政治和专业课，但是数学还是每天都要学，因为你会发现数学内容太多了，太容易忘了，我和我室友回寝室第一件事就是吐槽，”数学咋刚学就忘了啊“，所以还是要反复多次复习。专业课数据结构较为简单，照着网课学，理解透彻了不容易忘，但是操作系统内容太多，我的建议也是第一轮学习+做题用7天x12小时/天，第二轮开始记笔记以便之后的快速复习，第二轮花大概3天x12小时/天，第三轮照着你的笔记复习（我操作系统做了满满7页A4纸的笔记），只需要半天不到就可以完成一轮。

英语真题可以从十月份中旬开始做，这时候考纲词已经差不多过了一遍了，正确率也上来了，效率也高，没必要刚开始没背单词就去刷真题，吃力不讨好。如果自己阅读理解命中率还可以的话，比如20个错两到三个，那么只刷近五年真题就OK了，十年之前的题不值得刷。到十二月的时候，可以隔两天做一道阅读题，保持手感。作文自己总结记忆三篇左右不同类型的范文即可。

政治在十一月中旬之前，最好要把近代史的时间线和党史的时间线背下来（你可以花两三个晚上把黄皮书史纲过一遍，总结一下，同时也推荐空卡的史纲时间轴视频），这时大量的模拟卷横空出世，用刷题小程序（我用的是某盾，好像考完研被封了，上面有市场上几乎所有模拟卷的客观题，虽然也买了正版的肖四肖八，但感觉刷客观题没小程序方便）多次刷题，复刷错题，不会的就去冲刺背诵手册上找到对应知识点，记下来。

在冲刺阶段，数学要刷大量模拟题，并且建议刷到考试前一天，保持手感；专业课不用刷新题，保证每隔一段时间看看笔记和错题，没有遗忘即可；政治可以看看腿姐的时政总结，学一些大题的答题技巧。我考前两天做的事情：数学刷题+背诵专业课笔记+背诵肖四大题+记忆英语范文。

复试

分数不错的话提前联系老师挺重要的，很多老师会预先面试，虽然这个面试不会影响最终的录取结果。

复试流程

电科的线上复试流程是 英语自我介绍->英语问答（根据你的自我介绍）->专业课抽题（共享屏幕上有20个文档对应20道题目，自由选序号答题，你答完后，你抽的那道题就会被删掉，我抽到的是一道代码题，只需要讲算法思想）->专业课问答（按你的简历，比如我简历上写了计组92，就问了计组的题目）

提前退场不要紧张，我提前了5分钟退场，最终复试成绩还行。

写在最后

保持好的心态非常关键！整个考研期间，最迷惑的事情就是“怎么刚学完就忘了”，大家都是这样，不摆烂就是胜利！

frp用于内网的代理转发

frp

frp是一个常见的内网穿透工具。内网穿透，即 NAT 穿透，进行 NAT 穿透是为了使具有某一个特定源 IP 地址和源端口号的数据包不被 NAT 设备屏蔽而正确路由到内网主机。

UDP 内网穿透的实质是利用路由器上的NAT 系统。NAT 是一种将私有（保留）地址转化为合法IP地址的转换技术，它被广泛应用于各种类型 Internet 接入方式和各种类型的网络中。NAT可以完成重用地址，并且对于内部的网络结构可以实现对外隐蔽。

而在内网中，面对内网中不出网的机器，攻击者在外网无法直接访问到，通过frp进行代理，即可实现直接访问。

frp的工作模式

frp支持很多种协议，包括tcp、udp、http、https、socks等等，当然也可以实现端口对端口的转发。

frp分为客户端和服务器端，对应于不同的系统平台有不同的版本，但是名字都以frps和frpc进行区分，对应的配置文件有两种，分别是精简版和详细版配置文件：

Linux Namespace

Linux Namespace提供了一种内核级别隔离系统资源的方法，通过将系统的全局资源放在不同的Namespace中，来实现资源隔离的目的。不同Namespace的程序，可以享有一份独立的系统资源。目前Linux中提供了六类系统资源的隔离机制，分别是：

  • Mount: 隔离文件系统挂载点
  • UTS: 隔离主机名和域名信息
  • IPC: 隔离进程间通信
  • PID: 隔离进程的ID
  • Network: 隔离网络资源
  • User: 隔离用户和用户组的ID
  • Cgroup: 隔离进程组 (process groups) 所使用的物力资源 (如 cpu memory i/o 等等)

表中前六种Namespace正是实现容器所必须的隔离技术。

Docker环境下IPsec配置实验

选用Docker的原因

实验要求一台主机和四台虚拟机，硬件条件不允许，只好选用docker来代替。

Docker

第一次接触Docker，之前只知道Docker是一种容器化技术

Docker 和虚拟机的区别

同为虚拟技术，区别如下：

  • 容器和宿主机共享kernel，即宿主机的kernel就是容器的kernel
  • 虚拟机有独立的文件、进程、内存系统，而容器只是做了隔离，Docker是进程之间的隔离，而虚拟机可以实现系统级别的隔离
  • 容器管理和虚拟机管理在速度上有很明显的差距，虚拟机可以做到10台/min，而Docker容器的创建是秒级别的

Docker Engine

Docker Engine属于Docker的运行层。这是一套轻量化运行时及工具组合，负责管理容器、镜像、构建 等等。它以原生方式运行在Linux系统之上，并由以下元素构成：

  • Docker Daemon，运行在主机之上的程序
  • Docker Client，负责与Docker Daemon通信以执行命令，Docker采用的是CS模式
  • REST API，用于同Dockers Daemon远程交互

Docker 核心概念

Image 镜像

image 就类似虚拟机的镜像文件，可以用于创建容器

Container 容器

容器就是运行的单位，从镜像启动。Docker 会在镜像的最上一层创建一个可写层，镜像本身是只读的，保持不变

Repository 仓库

仓库是用来存放一类镜像的地方

Docker 四种网络模式（部分转载）

实现原理：

Docker使用Linux桥接，在宿主机虚拟一个Docker容器网桥(docker0，对应连接网卡是eth0），Docker启动一个容器时会根据Docker网桥的网段分配给容器一个IP地址，称为Container-IP，同时Docker网桥是每个容器的默认网关。因为在同一宿主机内的容器都接入同一个网桥，这样容器之间就能够通过容器的Container-IP直接通信（可以理解为一个公共网段，每个容器的eth0网卡都在网段上）。

Docker网桥是宿主机虚拟出来的，并不是真实存在的网络设备，外部网络是无法寻址到的，这也意味着外部网络无法通过直接Container-IP访问到容器。如果容器希望外部访问能够访问到，可以通过映射容器端口到宿主主机（端口映射），即docker run创建容器时候通过 -p 或 -P 参数来启用，访问容器的时候就通过[宿主机IP]:[容器端口]访问容器。

四类网络模式
Docker网络模式 配置 说明
host模式 –net=host 容器和宿主机共享Network namespace。
container模式 –net=container:NAME_or_ID 容器和另外一个容器共享Network namespace。 kubernetes中的pod就是多个容器共享一个Network namespace。
none模式 –net=none 容器有独立的Network namespace，但并没有对其进行任何网络设置，如分配veth pair 和网桥连接，配置IP等。
bridge模式 –net=bridge （默认为该模式）
host

如果启动容器的时候使用host模式，那么这个容器将不会获得一个独立的Network Namespace，而是和宿主机共用一个Network Namespace。容器将不会虚拟出自己的网卡，配置自己的IP等，而是使用宿主机的IP和端口。但是，容器的其他方面，如文件系统、进程列表等还是和宿主机隔离的。

使用host模式的容器可以直接使用宿主机的IP地址与外界通信，容器内部的服务端口也可以使用宿主机的端口，不需要进行NAT，host最大的优势就是网络性能比较好，但是docker host上已经使用的端口就不能再用了，网络的隔离性不好。

container

这个模式指定新创建的容器和已经存在的一个容器共享一个 Network Namespace，而不是和宿主机共享。新创建的容器不会创建自己的网卡，配置自己的 IP，而是和一个指定的容器共享 IP、端口范围等。同样，两个容器除了网络方面，其他的如文件系统、进程列表等还是隔离的。两个容器的进程可以通过 lo 网卡设备通信。

none

使用none模式，Docker容器拥有自己的Network Namespace，但是，并不为Docker容器进行任何网络配置。也就是说，这个Docker容器没有网卡、IP、路由等信息。需要我们自己为Docker容器添加网卡、配置IP等。

这种网络模式下容器只有lo回环网络，没有其他网卡。none模式可以在容器创建时通过–network=none来指定。这种类型的网络没有办法联网，封闭的网络能很好的保证容器的安全性。

bridge

当Docker进程启动时，会在主机上创建一个名为docker0的虚拟网桥，此主机上启动的Docker容器会连接到这个虚拟网桥上。虚拟网桥的工作方式和物理交换机类似，这样主机上的所有容器就通过交换机连在了一个二层网络中。

从docker0子网中分配一个IP给容器使用，并设置docker0的IP地址为容器的默认网关。在主机上创建一对虚拟网卡veth pair设备，Docker将veth pair设备的一端放在新创建的容器中，并命名为eth0（容器的网卡），另一端放在主机中，以vethxxx这样类似的名字命名，并将这个网络设备加入到docker0网桥中。可以通过brctl show命令查看。

bridge模式是docker的默认网络模式，不写–net参数，就是bridge模式。使用docker run -p时，docker实际是在iptables做了DNAT规则，实现端口转发功能。可以使用iptables -t nat -vnL查看。
